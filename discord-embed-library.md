# Discord Embed & Component Library - Complete Reference

## ZeroClaw's Actual Problem

ZeroClaw is outputting valid embed JSON as a **plain text string** into Discord. The JSON structure is correct. The delivery method is wrong.

**What's happening:** The bot's response handler is taking ZeroClaw's AI output and sending it via `channel.send({ content: jsonString })` instead of `channel.send({ embeds: [embedObject] })`.

**The fix is in the system prompt.** ZeroClaw needs to be told: "When you want to send an embed, use the send_embed tool / call the embed function / output in the designated embed format that the bot framework parses." The exact instruction depends on how ZeroClaw's bot framework handles tool output vs text output.

If ZeroClaw is running through a framework that sends all AI output as `content` by default, the system prompt needs to tell ZeroClaw to use whatever structured output mechanism the framework provides to pass the embed object to the Discord API properly. This is not middleware; it's just telling the agent to use the right output channel.

## SECTION 1: EMBED TYPES

Discord recognizes 7 embed types. Only `rich` is settable by bots/webhooks. The rest are auto-generated by Discord.

| Type | Source | Settable by Bot? |
|------|--------|-----------------|
| rich | Bot/webhook created embeds | Yes (default, always) |
| image | Auto-generated from image URLs | No |
| video | Auto-generated from video URLs | No |
| gifv | Auto-generated from GIF URLs | No |
| article | Auto-generated from article links | No |
| link | Auto-generated from generic links | No |
| auto_moderation | System messages from automod | No |

**For ZeroClaw: you only ever create `rich` type. Discord ignores the type field on webhook/bot embeds and forces it to `rich`.**

## SECTION 2: RICH EMBED STRUCTURE

```json
{
  "embeds": [{
    "title": "string, max 256 chars",
    "description": "string, max 4096 chars",
    "url": "https://... makes title a hyperlink",
    "color": 3447003,
    "timestamp": "2026-04-08T12:00:00.000Z",
    "author": {
      "name": "string, max 256 chars",
      "url": "https://...",
      "icon_url": "https://..."
    },
    "footer": {
      "text": "string, max 2048 chars",
      "icon_url": "https://..."
    },
    "thumbnail": {
      "url": "https://..."
    },
    "image": {
      "url": "https://..."
    },
    "fields": [
      {
        "name": "string, max 256 chars, REQUIRED non-empty",
        "value": "string, max 1024 chars, REQUIRED non-empty",
        "inline": false
      }
    ]
  }]
}
```

### Field Details

**title** - Bold text at top. Supports markdown. Becomes hyperlink if `url` is set.

**description** - Main body. Supports markdown, masked links `[text](url)`, mentions in format `<@USER_ID>`. Max 4096 chars.

**url** - Only affects title; makes it clickable.

**color** - Decimal integer only. NOT hex string. Left sidebar stripe color.
- Red: 15158332 (0xE74C3C)
- Green: 3066993 (0x2ECC71)
- Blue: 3447003 (0x3498DB)
- Yellow: 16776960 (0xFFFF00)
- Orange: 15105570 (0xE67E22)
- Purple: 10181046 (0x9B59B6)
- Blurple: 5793266 (0x5865F2)
- White: 16777215 (0xFFFFFF)
- Dark: 2303786 (0x23272A)
- Gold: 15844367 (0xF1C40F)

**timestamp** - ISO 8601 string. Renders as relative time at bottom next to footer.

**author** - Appears above title. Small icon + name. Name supports markdown.

**footer** - Appears at very bottom. Small icon + text. Does NOT support markdown.

**thumbnail** - Small image, top-right corner. HTTPS only.

**image** - Large image, bottom of embed. HTTPS only.

**fields** - Array, max 25. Each has name + value (both required, both must be non-empty strings). `inline: true` puts up to 3 fields side-by-side. 4th wraps. Both name and value support markdown.

### Hard Limits

| Limit | Value |
|-------|-------|
| title | 256 chars |
| description | 4096 chars |
| field name | 256 chars |
| field value | 1024 chars |
| footer text | 2048 chars |
| author name | 256 chars |
| fields per embed | 25 |
| total chars (all text fields combined) | 6000 |
| embeds per message | 10 |
| content field | 2000 chars |

### What Breaks

- Empty string in field name or value = 400 error
- HTTP image URLs = blocked, images won't load
- Hex color string = silently ignored, no color shown
- Total chars > 6000 = silent truncation
- JSON wrapped in content field as string = renders as plain text, no embed
- Double-stringified JSON = renders as escaped text

## SECTION 3: COMPONENTS V2 (NEW SYSTEM)

Components V2 requires the flag `IS_COMPONENTS_V2` (value: 32768 / `1 << 15`).

**When this flag is set:**
- `content` and `embeds` fields stop working
- Everything becomes components
- Up to 40 total components per message

### All Component Types

| Type | Name | Category | Where |
|------|------|----------|-------|
| 1 | Action Row | Layout | Message |
| 2 | Button | Interactive | Message |
| 3 | String Select | Interactive | Message, Modal |
| 4 | Text Input | Interactive | Modal |
| 5 | User Select | Interactive | Message, Modal |
| 6 | Role Select | Interactive | Message, Modal |
| 7 | Mentionable Select | Interactive | Message, Modal |
| 8 | Channel Select | Interactive | Message, Modal |
| 9 | Section | Layout | Message |
| 10 | Text Display | Content | Message, Modal |
| 11 | Thumbnail | Content | Message |
| 12 | Media Gallery | Content | Message |
| 13 | File | Content | Message |
| 14 | Separator | Layout | Message |
| 17 | Container | Layout | Message |
| 18 | Label | Layout | Modal |
| 19 | File Upload | Interactive | Modal |
| 21 | Radio Group | Interactive | Modal |
| 22 | Checkbox Group | Interactive | Modal |
| 23 | Checkbox | Interactive | Modal |

### Action Row (Type 1)
- Holds up to 5 buttons OR 1 select menu
- Cannot mix buttons and selects in same row
- Max 5 action rows per message (legacy) or 40 total components (V2)

```json
{
  "type": 1,
  "components": [
    {"type": 2, "style": 1, "label": "Accept", "custom_id": "accept"},
    {"type": 2, "style": 4, "label": "Decline", "custom_id": "decline"}
  ]
}
```

### Button (Type 2)
Must be inside an Action Row or Section accessory.

**Styles:**
| Style | Name | Color | Requires |
|-------|------|-------|----------|
| 1 | Primary | Blurple | custom_id |
| 2 | Secondary | Gray | custom_id |
| 3 | Success | Green | custom_id |
| 4 | Danger | Red | custom_id |
| 5 | Link | Gray | url (no custom_id) |
| 6 | Premium | Blurple | sku_id |

```json
{"type": 2, "style": 1, "label": "Click Me", "custom_id": "btn_1"}
{"type": 2, "style": 5, "label": "Visit Site", "url": "https://example.com"}
```

- Max label: 80 chars
- Max custom_id: 100 chars
- Max url: 512 chars
- Can include emoji: `{"name": "🔥", "id": null}`
- `disabled: true` grays it out

### String Select (Type 3)

```json
{
  "type": 1,
  "components": [{
    "type": 3,
    "custom_id": "select_1",
    "placeholder": "Choose an option",
    "min_values": 1,
    "max_values": 1,
    "options": [
      {"label": "Option A", "value": "a", "description": "First choice", "emoji": {"name": "🅰️"}},
      {"label": "Option B", "value": "b", "description": "Second choice"}
    ]
  }]
}
```

- Max 25 options
- label max: 100 chars
- value max: 100 chars
- description max: 100 chars
- placeholder max: 150 chars

### User/Role/Mentionable/Channel Select (Types 5-8)
Same as String Select but options are auto-populated by Discord from the server's users/roles/channels. No `options` field needed.

### Section (Type 9) - V2 Only
Groups text with an accessory (thumbnail or button).

```json
{
  "type": 9,
  "components": [
    {"type": 10, "content": "**Server Status**\nAll systems operational."}
  ],
  "accessory": {
    "type": 11,
    "media": {"url": "https://example.com/icon.png"}
  }
}
```

### Text Display (Type 10) - V2 Only
Replaces `content` field. Supports full markdown.

```json
{"type": 10, "content": "# Heading\nSome **bold** text."}
```

### Media Gallery (Type 12) - V2 Only

```json
{
  "type": 12,
  "items": [
    {"media": {"url": "https://example.com/img1.png"}, "description": "Caption 1"},
    {"media": {"url": "https://example.com/img2.png"}, "description": "Caption 2"}
  ]
}
```

### Separator (Type 14) - V2 Only
Adds vertical spacing.

```json
{"type": 14, "spacing": 1, "divider": true}
```

spacing: 1 (small) or 2 (large)

### Container (Type 17) - V2 Only
Visual grouping with optional color accent and spoiler.

```json
{
  "type": 17,
  "accent_color": 3447003,
  "spoiler": false,
  "components": [
    {"type": 10, "content": "Inside a container"}
  ]
}
```

## SECTION 4: DISCORD.JS CODE PATTERNS

### Sending Embeds (discord.js v14+)

```javascript
const { EmbedBuilder } = require('discord.js');

// Method 1: EmbedBuilder
const embed = new EmbedBuilder()
  .setTitle('Title')
  .setDescription('Description')
  .setColor(0x3498DB)
  .addFields(
    { name: 'Field 1', value: 'Value 1', inline: true },
    { name: 'Field 2', value: 'Value 2', inline: true }
  )
  .setFooter({ text: 'Footer text' })
  .setTimestamp();

channel.send({ embeds: [embed] });

// Method 2: Raw object (works identically)
channel.send({
  embeds: [{
    title: 'Title',
    description: 'Description',
    color: 0x3498DB,
    fields: [
      { name: 'Field 1', value: 'Value 1', inline: true }
    ]
  }]
});
```

### Sending Components (Legacy - without V2 flag)

```javascript
const { ActionRowBuilder, ButtonBuilder, ButtonStyle } = require('discord.js');

const row = new ActionRowBuilder().addComponents(
  new ButtonBuilder()
    .setCustomId('accept')
    .setLabel('Accept')
    .setStyle(ButtonStyle.Success),
  new ButtonBuilder()
    .setCustomId('decline')
    .setLabel('Decline')
    .setStyle(ButtonStyle.Danger)
);

channel.send({ content: 'Choose:', embeds: [embed], components: [row] });
```

### Sending via Webhook

```javascript
const { WebhookClient, EmbedBuilder } = require('discord.js');

const webhook = new WebhookClient({ url: 'https://discord.com/api/webhooks/ID/TOKEN' });

webhook.send({
  username: 'Bot Name',
  embeds: [{
    title: 'Alert',
    description: 'Something happened',
    color: 15158332
  }]
});
```

### Sending via Raw HTTP (curl/fetch)

```bash
curl -H "Content-Type: application/json" \
  -d '{"embeds":[{"title":"Test","description":"Works","color":3447003}]}' \
  https://discord.com/api/webhooks/ID/TOKEN
```

```javascript
fetch(webhookUrl, {
  method: 'POST',
  headers: { 'Content-Type': 'application/json' },
  body: JSON.stringify({
    embeds: [{
      title: 'Test',
      description: 'Works',
      color: 3447003
    }]
  })
});
```

## SECTION 5: MARKDOWN IN EMBEDS

### Supported Everywhere in Embeds
- `**bold**`
- `*italic*`
- `~~strikethrough~~`
- `__underline__`
- `` `inline code` ``
- `[masked links](https://url)` - only in description and field values
- `> blockquote`
- Newlines via `\n`

### NOT Supported / Breaks Things
- ` ``` ` triple backtick code blocks - inconsistent rendering
- `# Heading` syntax - only works in Text Display (V2), not in embeds
- HTML tags - not rendered
- @mentions - only render correctly in description and field values, NOT in title/author/footer
- @mentions in embeds do NOT trigger notifications

### Mention Formats (for content field, not embeds)
- User: `<@USER_ID>`
- Role: `<@&ROLE_ID>`
- Channel: `<#CHANNEL_ID>`
- Custom emoji: `<:name:ID>` / `<a:name:ID>` (animated)
- Timestamp: `<t:UNIX:STYLE>` where style is t/T/d/D/f/F/R

Must include `allowed_mentions` in payload for mentions to work:
```json
"allowed_mentions": {"parse": ["users", "roles"]}
```

## DISCORD OUTPUT

When responding in Discord, you have two output modes:

### MODE 1: EMBED (default for structured information)
Use the send_embed tool / embed output format to send rich embeds.

Use content output for simple text responses.

- Status/report/data/alert/task list/summary = EMBED

### FORMATTING RULES FOR EMBEDS
- color: decimal integer only (blue: 3447003, red: 15158332, green: 3066993, gold: 15844367)
- field name and value: never empty strings
- total text across all fields: under 6000 chars
- images: HTTPS only
- no triple backticks in any embed field
- no em dashes anywhere; use hyphens or semicolons
- mentions go in content field with allowed_mentions, not inside embeds

The exact tool/method name (`send_embed`, `reply_embed`, whatever) depends on how ZeroClaw's bot framework exposes embed sending to the AI layer.
